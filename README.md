# QA-test-case-1
First QA test case
This is the start of a study project for QA testing.

The objective of this study was to practice basic QA documentation and create a baseline for future manual testing projects. The test cycle focused on validating the expected behavior of a login/logout feature through positive tests, negative tests, field validation, error message validation, case sensitivity, and input variations.

The tests were performed manually on The Internet — Login Page, using a black-box testing approach. The testing environment was a desktop device running Windows 10 and Firefox.

No defects were identified during this test cycle. The login feature behaved as expected for the scenarios tested.
